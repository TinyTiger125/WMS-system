# 跨境进销存系统 SOP（上线交付版）

## 1. 业务规则与流程控制
- 采购单确认前必须有供应商、明细行、有效数量。
- 销售单确认前必须有客户、明细行、有效数量。
- 采购单/销售单/库存单取消仅允许老板角色。
- 出库过账默认禁止负库存（老板可用于应急处理）。

## 2. 库存控制
- 商品支持设置 `最低库存预警` 与 `最高库存预警`。
- 系统每小时刷新库存预警清单。
- 菜单：`跨境进销存 -> 进销存流程 -> 8. 库存预警`。

## 3. 价格与成本追踪
- 自动记录销售价、成本价、申报价变更历史。
- 菜单：`跨境进销存 -> 经营报表 -> 价格变更记录`。

## 4. 报表与看板
- 系统每日自动生成经营日报（采购、销售、出入库、库存预警汇总）。
- 菜单：`跨境进销存 -> 经营报表 -> 经营日报`。

## 5. 报关导出
- 导出前自动校验商品报关资料完整性（HS Code、报关名、申报价、产地、净毛重）。
- 不完整时阻止导出并给出缺失明细。
- 菜单：`跨境进销存 -> 经营报表 -> 报关导出`。

## 6. 权限与审计
- 关键动作（商品修改、采购确认/取消、销售确认/取消、出入库过账/取消、报关导出）写入审计日志。
- 菜单：`跨境进销存 -> 经营报表 -> 操作审计日志`（老板/系统管理员可见）。

## 7. 备份与恢复演练
- 备份：`./scripts/backup_odoo_db.sh`
- 恢复：`./scripts/restore_odoo_db.sh /path/to/backup.dump`
- 演练检查：`./scripts/check_backup_restore.sh`

## 8. 交付封装检查清单
- 顶级应用入口仅保留 `跨境进销存`。
- 菜单已按业务动线重排。
- 全部核心操作页面中文化。
- 生产上线前需执行一次 UAT 全链路：
  1. 商品登记
  2. 采购下单
  3. 采购入库
  4. 销售下单
  5. 销售出库
  6. 报关导出
